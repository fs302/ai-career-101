Runtime Flow

从职业配置到可评估的职业导师

01 职业 YAML

身份、边界、规则、工作流、工具和 benchmark 权重。

→
02 Workflow Engine

处理输入与附件，按角色声明调用工具并组装 prompt。

→
03 Tool Registry

图片理解、视频提示、中文英译、MiniMax 语音合成。

→
04 Model Router

按请求选择 minimax、GLM、Qwen、DeepSeek 等模型，避免共享状态污染。

→
05 Benchmark

用本地 case 评估职业完成度、人类对齐度和工具调用质量。

Tools + Skills

用工具和职业技能组合出可工作的职业导师

查看工具 API
vision.describe 图片理解

把空间、餐食、作品、眼部照片等视觉输入转成职业上下文。

translation.zh_en 中英转换

服务同传和字幕翻译，支持语义压缩、语气转换和口语表达。

image.generate 图片生成

为插画、游戏美术、室内设计和广告策划生成概念图或视觉草案。

speech.tts 语音合成

调用 MiniMax CLI，把口译稿生成可播放的英文音频。

attachment.video_notice 视频附件处理

接收视频素材并进入工作流，当前阶段先做附件提示和文字协作。

attachment.file_notice 通用附件处理

记录非图片/视频附件，为后续 PDF、表格、文档工具保留入口。
